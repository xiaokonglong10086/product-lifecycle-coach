# 独立产品经理的 Codex 开发与外包判断

本文件用于决定一项工作应该由用户自己完成、交给 Codex，还是交给外包工程师。

## 1. 总原则

判断标准不是“会不会写代码”，而是：
- 错了会造成什么后果；
- 是否涉及真实客户数据和生产环境；
- 是否需要长期维护；
- 是否需要专业安全、性能或架构责任；
- 是否能用明确验收标准验证。

## 2. 用户可以直接负责的工作

通常由产品经理完成：
- 用户、客户和业务调研；
- 需求收敛、优先级和产品决策；
- 概念方案、流程、原型和文案；
- PRD、AI 功能规格和验收标准；
- Mock 数据和演示内容；
- Codex 任务拆解和结果评审；
- 前端 Demo 的体验验收；
- 客户试点方案、培训和反馈；
- 是否上线、是否扩大范围的产品判断。

## 3. 适合 Codex 独立执行的工作

前提：范围清楚、可本地验证、失败可恢复。

- React/TypeScript 前端页面和组件；
- Mock 状态、路由和交互；
- 简单数据转换、校验脚本；
- 现有代码中的小步重构；
- UI 适配、响应式和可访问性修复；
- 单元测试、组件测试和构建修复；
- 简单本地存储或演示后端；
- 已明确接口的前端接入；
- 文档、类型和测试补齐；
- 可丢弃的原型验证。

每次任务必须包含：当前问题、本轮目标、修改范围、禁止项、必须完成、验收标准、测试命令、修改文件说明。

## 4. Codex 可以做，但需要技术人员复核

- 简单 REST API 和常规 CRUD；
- 数据库表初稿；
- 身份系统的非核心页面接入；
- 文件上传、转码或异步任务原型；
- 模型 API、RAG 或工具调用的原型；
- Docker、CI/CD 初稿；
- 中等规模状态管理和缓存；
- 生产系统的局部重构。

这里的技术人员至少要检查：安全、数据、并发、失败恢复、部署和维护成本。

## 5. 默认应找外包工程师的工作

出现以下任一项，不应只凭 Codex 原型直接上线：
- 登录、鉴权、角色权限和越权风险；
- 多租户数据隔离；
- 员工、客户、录音、简历、作业等敏感数据；
- 正式数据库、备份、迁移和灾难恢复；
- SSO、企业微信、钉钉、LDAP 等企业集成；
- 支付、合同、开票或真实权益核销；
- 高并发、实时通信、消息队列和异步任务；
- 文件转码、病毒扫描和大文件存储；
- 生产部署、监控、告警、日志和成本控制；
- 安全审计、渗透测试和合规要求；
- AI 自动写入正式记录、自动评分影响人员、自动执行不可逆动作；
- 需要服务等级承诺的客户生产系统。

## 6. 五问判断法

1. 失败是否会泄露数据、损失资金、影响员工或客户权益？
2. 是否需要稳定运行超过一个 Demo/试点周期？
3. 是否有多个用户、组织或并发操作？
4. 是否涉及不可逆数据修改或外部系统？
5. 用户是否无法通过页面验收发现底层错误？

- 0-1 个“是”：通常可由用户 + Codex 完成；
- 2-3 个“是”：Codex 实现，技术复核；
- 4-5 个“是”：外包工程师负责正式实现。

## 7. 外包前最小交付包

产品经理不需要先写完整技术架构，但至少提供：
- 一页业务背景和本期目标；
- 已确认 PRD；
- 核心流程和页面；
- 角色与权限；
- 关键数据含义；
- AI 功能边界；
- 当前代码和可保留资产；
- 明确不做；
- 验收场景；
- 上线环境、数据和安全要求；
- 外包需要提供的技术方案、排期、风险和维护说明。

## 8. Codex 任务完成门槛

Codex 完成后必须提供：
- 修改文件列表；
- 关键实现说明；
- 运行的测试和构建命令；
- 通过/失败结果；
- 未解决问题；
- 是否改动了未授权范围；
- 手工验收路径。

产品经理按真实用户路径验收，不以“代码写完”作为完成。

## 9. 先看产品做到哪一步

- **可交互 Demo：** 你 + Codex 可以承担绝大多数工作；允许 Mock，但必须明确标注。
- **内部可用版：** Codex 可实现，涉及真实数据、接口和部署时安排技术复核。
- **客户试点版：** 你负责产品、实施和验收；权限、敏感数据、正式数据库、部署和监控必须有工程责任人。
- **正式生产版：** 不能只靠“Codex 跑通”；需要可追责的工程实现、运维、安全和恢复。

使用 `prototype-to-production.md` 判断升级条件。

## 10. 外包合作中的产品经理控制点

外包不是把需求扔出去。至少要控制：
- 业务目标和范围；
- 必须保留的代码和体验；
- 分阶段可运行交付；
- 验收标准和复现步骤；
- 源码、仓库、账号、密钥和云资源归属公司；
- 数据、开源许可证和知识产权；
- 部署、备份、恢复、监控和维护文档；
- 质保期、故障响应和后续维护成本。

使用 `assets/templates/OUTSOURCE_BRIEF.md`，并运行 `scripts/audit_outsource_brief.py`。
