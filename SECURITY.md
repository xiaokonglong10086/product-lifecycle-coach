# Security policy

Do not report credentials, customer data, private PRDs, recordings, resumes, or other sensitive information in public issues.

For a suspected vulnerability, open a minimal issue that contains no sensitive exploit data and ask the maintainer for a private contact channel. Rotate any exposed credential immediately.

This Skill is not a substitute for production security review. Authentication, authorization, multi-tenant isolation, sensitive data handling, production databases, deployment, monitoring, backup, and disaster recovery require accountable engineering review.
