# 产品实现 QA 核心规范

审查真实产品行为，不审查 Codex 的完成说明。开始前读取设计理念、PRD、工程与 AI 约束、模拟数据、实际代码与运行页面、功能矩阵和决策记录。

能够运行时必须安装并启动，执行类型检查、测试和构建，检查目标桌面与移动端，逐页逐按钮逐状态操作，验证刷新恢复和跨端同步。不能运行时明确证据限制。

问题分类包括阻塞、理念违背、PRD 不符、L3/L4 不足、体验问题、视觉问题、模拟数据问题和生产差距。

固定检查无响应按钮、Toast 代替流程、TODO、空白占位、加载/空/失败/重试/取消/撤销/刷新恢复、统一模拟业务内核、多端联动、来源冲突和功能矩阵一致性。QA 不得直接修改 PRD。
