# 黄金 PRD 表达模式

文档从当前产品开始。首页只说明产品名称、文档状态、负责人、适用端、用途和少量最高优先级产品规则，不写版本对比和生成过程。

阅读顺序应形成完整产品画面：产品概述；结构与页面关系；完整产品流程和用户流程；内部运作；通用交互；按真实任务展开功能；跨功能规则、异常、质量、非功能和待确认。

三类图分工明确：页面关系图说明跳转；完整产品流程覆盖主任务和失败恢复；内部运作图说明任务登记、异步、AI、待确认、人工确认、正式写入和质量回流。

复杂功能先给页面布局和整体流程，再逐项展开。表格用于对照，连续机制使用短段落和步骤。异常必须说明用户看到什么、还能做什么、已完成内容是否保留、如何重试、是否阻止正式结果和最终状态。
