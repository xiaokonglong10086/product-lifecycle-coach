# 产品功能 PRD 输出合同

主 PRD 是当前产品事实说明书，不是版本报告、项目计划、教练复盘或工程接口文档。

默认标题：`[产品名称] 产品功能需求文档`。

读者应能回答：产品解决什么问题；谁在什么场景使用；产品由哪些页面和模块组成；如何完成任务；失败如何恢复；内部如何登记、处理、确认、写入和回流；页面展示什么；状态如何变化；结果去哪里；怎样验收。

简洁只删除版本过程、教练分析、重复、空泛价值和工程噪音。必须保留页面布局、字段、步骤、状态、异常、恢复、数据去向、AI 边界、交互反馈、跨端影响和验收。

正式文档禁止出现产品教练、PRD Vx、V0.x、P0/P1、基线、本版、上一版、版本变化、第 0 章、FRD、开发批次、路线图和生命周期治理等过程语言。
